Section 01

What Is VaultArc Learn?

A personal AI-powered study app. You write or upload your notes, and the app uses AI to help you understand, retain, and test yourself — going beyond passive reading into active learning.

Core Philosophy

Active Recall Over Passive Reading
Every AI feature pushes you to retrieve and apply knowledge, not just re-read. Quizzes, flashcards, and Teach It Back force active engagement with the material.
Understanding Over Memorisation
AI prompts are written to explain WHY and HOW, use real-world analogies, and target common misconceptions — not just list definitions.
Spaced Repetition Built In
Flashcards use the SM-2 algorithm. Cards you struggle with come back sooner. Cards you know well are spaced further apart. Scientifically proven to improve long-term retention.
Your Notes, Your AI
Every AI feature operates on your actual note content — quizzes, summaries, mind maps are all generated from what you wrote, not generic internet knowledge.

Who It's For

Anyone studying structured content — certifications (CompTIA, Microsoft, Cisco), academic subjects, professional development. Single-user personal vault.

Access

URL: learn.vaultarc.co.za
Protected by Cloudflare Access (login wall — you authenticate before the page loads). All API requests require an x-api-key header. The key is set once in environment variables and never exposed in source code.

File Storage

Cloudflare R2
S3-compatible object storage. Stores raw uploaded PDF files. 10GB free tier. Files deleted when note is deleted.

Vector Search

Cloudflare Vectorize
768-dim cosine similarity index. Powers semantic search and related-note connections. Auto-indexed on create/update.

AI — Reasoning

Llama 3.3 70B
Used for: quiz, scenario, ask AI, teachback, priority, mindmap, flashcards. Best quality.

AI — Fast

Llama 3.1 8B
Used for: summary, ELI5, key terms. Faster and cheaper — sufficient for extraction tasks.

Embeddings

BGE Base EN v1.5
Generates 768-dim vectors for Vectorize. First 2,000 chars used. Called on every note create/update.

Content Truncation (truncateForAI)

Before any AI call, note content is capped at 20,000 characters. If longer, it takes the first 10,000 chars + last 10,000 chars with a [... content truncated ...] marker — so the AI always sees the beginning and end of the note.

Section 04

Database Schema

Cloudflare D1 (SQLite). Four tables. All IDs are nanoid strings (21 chars). All timestamps are Unix milliseconds (Date.now()).

-- Core note storage CREATE TABLE notes ( id TEXT PRIMARY KEY, title TEXT NOT NULL, content TEXT NOT NULL DEFAULT '', source_type TEXT NOT NULL DEFAULT 'text', -- 'text' | 'upload' file_key TEXT, -- R2 object key (only for uploads) subject_id TEXT, -- FK → subjects.id (nullable) section TEXT, -- section name within subject position INTEGER, -- sort order tags TEXT DEFAULT '', -- comma-separated string created_at INTEGER NOT NULL, updated_at INTEGER NOT NULL ); -- Subject folders CREATE TABLE subjects ( id TEXT PRIMARY KEY, name TEXT NOT NULL, color TEXT NOT NULL DEFAULT '#b87040', created_at INTEGER NOT NULL ); -- Generated quizzes (questions stored as JSON string) CREATE TABLE quizzes ( id TEXT PRIMARY KEY, note_id TEXT NOT NULL, title TEXT NOT NULL, questions TEXT NOT NULL, -- JSON: [{question, options[], correct, explanation}] created_at INTEGER NOT NULL ); -- Quiz attempt results CREATE TABLE quiz_results ( id TEXT PRIMARY KEY, quiz_id TEXT NOT NULL, score INTEGER NOT NULL, total INTEGER NOT NULL, answers TEXT NOT NULL, -- JSON: [0, 2, 1, 3, ...] taken_at INTEGER NOT NULL );

What's NOT in D1

Flashcard decks, chat history, summaries, ELI5, key terms, priority results, and mind maps are all cached in localStorage in the browser — not in the database. This keeps the worker fast and the database lean.

Section 06

AI Features & Prompts

Every feature has a carefully engineered system prompt. The philosophy: force genuine understanding, not surface recall. Prompts resist generic, hollow output.

Summarise Llama 8B

Four named sections — not a flat bullet list. Builds a mental model, doesn't just compress text.

You are an expert tutor who understands how people learn and retain information. Summarise the notes in a way that builds genuine understanding — not just a list of facts. Structure: **The Big Picture** — one sentence on what this topic is really about and why it matters **Core Concepts** — bullet points: concept → what it does / why it exists (not just a definition) **How It Fits Together** — 2-3 sentences connecting the concepts to each other **Common Confusion** — one thing students typically misunderstand about this topic Write as if explaining to someone who wants to truly understand, not just memorise.

Simplify / ELI5 Llama 8B

Opens with a vivid analogy, closes with one sticky sentence. Not dumbed down — made genuinely clear.

You are a brilliant teacher who can make any complex topic click instantly. Your goal is not to dumb things down — it is to build genuine understanding through analogy and story. Rules: - Open with one powerful analogy that captures the whole topic (vivid and concrete) - Explain each key concept: "what is this LIKE in real life?" — use things everyone has experienced - Short sentences. Bold the concept when you first introduce it. - Point out what makes this topic tricky — where people get confused and why - Close with: "**The one thing to remember:** ..." — one sentence that will stick Never say "imagine" as your first word. Start with the analogy directly.

Key Terms Llama 8B

Only load-bearing terms — concepts that unlock understanding of other things. Max 12. Trivia filtered out.

Extract the key terms that are load-bearing to understanding this topic. Only include terms where knowing them unlocks understanding of something else. Skip supporting details and examples. Prioritise terms that are frequently tested, commonly confused, or essential building blocks. Output ONLY a JSON array of strings. Maximum 12 terms. Short phrases only (1-4 words each).

What Matters (Priority) Llama 70B

Ruthless triage into three buckets. Each item has a specific WHY and a real-world ANCHOR to make it stick.

You are a master tutor who knows exactly what to focus on to build understanding efficiently. Analyse these notes and ruthlessly prioritise what the student needs to know. Buckets: - must_know: 2-4 concepts that are the foundation everything else builds on. If you had one hour to study, spend it here. - should_know: 3-5 concepts that deepen understanding. Important, but only after must_know is solid. - nice_to_know: interesting details, edge cases — valuable but skippable under time pressure. For each concept: - concept: short name (3-6 words max) - why: one sentence — WHY this belongs at this priority level (not just "it's important") - anchor: a concrete real-world analogy that makes this concept stick (1-2 sentences)

Mind Map Llama 70B

Exactly 3 branches, 2 children each. Labels kept short for SVG nodes. Frontend renders center + branches only.

Pick only the TOP 3 most important concept areas — ignore minor details. - center: the main topic (3 words max) - branches: EXACTLY 3 branches — the 3 most critical concept areas - each branch has EXACTLY 2 children — the 2 most important sub-points - keep all labels SHORT (2-4 words max) - ruthlessly prioritise what matters most

Ask AI — RAG Llama 70B + BGE embed

Questions answered in the context of the selected note. Vectorize simultaneously pulls up to 2 related notes (score >0.65) as extra context — the answer can draw on knowledge from across the whole vault.

System prompt: "You are a helpful study assistant. The user is studying notes titled '{title}'. Answer questions based on the note content. If the answer isn't in the notes, say so. NOTE CONTENT: {noteText} RELATED NOTES (for additional context): {extraContext}"

Connections BGE embed only — zero LLM tokens

Embeds the note's title + first 2,000 chars, queries Vectorize topK=5, filters score >0.7. Returns matching note titles. No AI generation at all — pure vector similarity.

Teach It Back Llama 70B

Two-step Feynman technique. Step 1 generates a question. Step 2 evaluates the student's explanation for actual understanding vs word repetition.

Question generation: Pick the single concept that is most important AND most commonly misunderstood. Ask the student to explain it as if teaching someone who has never encountered the topic. The question must force them to explain the WHY and HOW, not just the WHAT. Evaluation — scoring criteria: "strong": explained WHY and HOW correctly. They could teach it. "partial": right idea but missed something important, or explained WHAT without WHY "needs_work": confused, used right words incorrectly, or mostly missed the point Returns: { score, got_right, missed, remember }

Flashcards Llama 70B

Cards test understanding, not rote recall. Front asks why/how/difference. Back gives answer + analogy if helpful. One idea per card.

Front: a question that tests understanding, not just recall. Ask "why", "how", "what happens when", "what is the difference between" — not "what is X" Back: clear, complete answer in plain language. Include analogy/example in one sentence if useful. One idea per card. Never cram two concepts into one. Prioritise concepts that are commonly confused, frequently tested, or foundational.

Quiz — Standard Llama 70B

4-option multiple choice. Wrong answers are real misconceptions — not obviously wrong. Explanations teach the reasoning.

Test comprehension and application — "why does X happen", "what would you do if", "what is the difference between" — not "what is the definition of" Wrong answers must be plausible — common misconceptions, related concepts that sound right. The explanation must teach: WHY the correct answer is right AND why the most tempting wrong answer is wrong.

Quiz — Scenario Llama 70B

Applied thinking. Every question describes a realistic situation first — troubleshooting, incident response, design decisions. Forces application of knowledge, not recall.

Every question must describe a specific, realistic situation — a problem to solve, an incident to respond to, a decision to make. The correct answer requires understanding WHY, not just WHAT. Wrong answers must represent real mistakes: the "almost right" choice, the "I'd do that first" trap. Contexts: troubleshooting, incident response, design decisions, policy choices — whatever fits.
Section 07

Spaced Repetition System

Built on the SM-2 algorithm. Runs entirely in the browser — no server, no database calls. Card state lives in localStorage. Each note has its own independent deck.

How SM-2 Works

After reviewing each card you rate it Hard / OK / Easy (1/2/3). The algorithm adjusts the interval (days until next review). Cards you find easy get pushed further out. Hard cards come back tomorrow. Over time, you spend almost zero time on things you already know.

Card State Object
type CardState = { front: string back: string interval: number // days until next review easeFactor: number // starts at 2.5 dueDate: number // Unix ms timestamp reps: number // times seen }
Rating Logic
Hard (1): interval = 1 day, reps reset to 0 OK (2): interval = prev × easeFactor Easy (3): interval = prev × easeFactor easeFactor += 0.1 (max 2.5) All: easeFactor = max(1.3, ef - 0.1 + 0.1×rating) Next review = now + interval × 86,400,000ms

New Card Defaults

Every new card: interval: 1, easeFactor: 2.5, dueDate: now (immediately due), reps: 0. First Easy rating schedules it 4 days out. First OK schedules it 1 day out.

Section 09

Security

Two independent layers. Either alone would be sufficient for a personal tool — both together mean nothing reaches your data unless it came from you through your browser.

Layer 1 — Cloudflare Access (Zero Trust)
The frontend at learn.vaultarc.co.za is behind Cloudflare Access. Any request triggers an auth check. You log in once (email OTP or identity provider) — Cloudflare issues a signed JWT cookie. Without it, you see the CF login screen, not the app. Blocks bots, scrapers, and anyone who doesn't have credentials.
Layer 2 — API Key (x-api-key header)
Every API request includes an x-api-key header. The worker checks it against env.API_KEY (a Wrangler secret). If missing or wrong → 401 Unauthorized before any route handler runs. The key is stored as a Cloudflare Pages env var — never in source code. OPTIONS preflight requests are exempt so CORS works.

CORS Policy

Worker allows three origins: learn.vaultarc.co.za (production), vaultarc-learn.pages.dev (preview), and localhost:5173 (local dev). All other origins are blocked at the CORS layer.

What Is NOT Stored or Transmitted

  • No passwords — Cloudflare Access handles identity
  • No analytics or third-party scripts
  • Note content never leaves Cloudflare infrastructure — Workers AI runs on CF's own hardware
  • R2 files are private — no public URLs generated
  • Single-tenant — no multi-user data separation needed, no user accounts
